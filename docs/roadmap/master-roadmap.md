# Master Roadmap

This roadmap consolidates the shared planning session into canonical phases.

## Phase 1: Local Agent Foundation

- Project scanner.
- Workspace sandbox.
- Local config and logger.
- Offline policy checks.
- CLI foundation.

## Phase 2: Coding Knowledge DB Foundation

- SQLite schema.
- CLI and localhost API.
- Seed loader.
- Knowledge item, error pattern, code example, and fix recipe tables.

## Phase 3: Ingestion and Search

- Raw to processed to reviewed to approved to indexed pipeline.
- Validation and secret scanner.
- Duplicate candidate detection.
- Ranked search by keyword, error, framework, tag, and confidence.
- Ingest reports in Markdown and JSON.

## Phase 4: Initial Offline Knowledge Pack

- At least 100 knowledge items.
- At least 50 error patterns.
- At least 50 fix recipes.
- At least 50 code examples.
- At least 30 QA/security checklist items.
- Approved local runtime data with no secret or unsafe command findings.

## Phase 5: Advanced Coding Intelligence

- Root cause engine.
- Error cluster engine.
- Fix recommendation engine.
- Dependency analyzer.
- Project fingerprint engine.
- Framework rule engine.
- Regression risk analyzer.
- Learning from local QA reports.
- Knowledge graph.

## Phase 6: Engineering Accounting DB

- SQLite accounting DB.
- Resource monitor.
- Patch ledger.
- QA ledger.
- Rollback ledger.
- Model runtime tracker.
- Immutable audit logs.
- Local API and dashboard.

## Phase 7: Accounting Analytics and Governance

- Project health score.
- Bug cost analyzer.
- Model efficiency analyzer.
- Failure forecast engine.
- Regression predictor.
- Resource optimization advisor.
- Event bus.
- Policy engine.
- Backup/restore.

## Phase 8: Ecosystem Dashboard

- CEO summary.
- Developer detail.
- System health.
- QA trends.
- Risky patch view.
- Resource pressure.
- Model comparison.
- Reports export.

## Future Expansion

- Marketing Knowledge DB as a separate local/offline domain brain.
- Multi-project local registry.
- Cross-project fix learning.
- Windows-first resource collectors.
- Richer VS Code integration.
- Deployment readiness gates.
- Local event replay for audit and debugging.
